Different Grid Sizes

Dont forget, we're building mobile first.

We have the option of using 3 different grid-sizes. Using .large-X .columns will result in columns stacking below 1280px, using .medium-x columns will result in columns stacking below 768px, and using .small-x columns will keep column widths even down to devices under 768px. Building semantic grids

Man, how many class names does one person need? You might be dreaded about the potential of a div looking like this:

div.large-3.medium-5.small-3.large-offset-3.small-centered.push-8.pull2 Okay that might be a bit overexaggeraded , but we hope you get the point

We'll we're here to save the day! By using our grid mixin functions, you can avoid cluttering your elements with layout class names and focus on using SEMANTICALLY CORRECT elements - omnomnom

So, layout your elements, ie.:

<div class="container">
 <section>
  <hgroup>

  </hgroup>
  <article>

  </article>
  <aside>

  </aside>
  <footer>

  </footer>
 </section>
</div>

Use the semantic grid in your scss files like this:


.container {
 section {
  @include grid-row();
  hgroup {
   @include grid-column(12);
  }
  article {
   @include grid-column(8);
  }
  aside {
   @include grid-column(4);
  }
  footer {
   @include grid-column(12);
  }
 }
}

Rejoyce, no bloated class naming!
